## Rationale and tradeoffs

- **Next.js + TypeScript:** One framework for UI + API routes, strong ecosystem, fits “shareable deployed app” and preview deploys on Vercel; typing supports strict trip schemas.
- **Tailwind:** Fast UI iteration, consistent spacing/typography tokens, works well with component-driven structure; tradeoff is utility-heavy markup vs. abstracted design systems (mitigated by shared components and tokens).
- **OpenAI:** Widest adoption, mature HTTP APIs, tooling, and docs; good fit for **reliable structured outputs** when combined with schema validation. Tradeoffs: cost at scale, vendor lock-in, policy/rate limits—mitigated by isolating the provider behind one module and revisiting in TC-040 (fallback, cost caps).
- **Client-only persistence:** Lowest operational burden for MVP (no DB provisioning, migrations, or auth). Tradeoff: no cross-device sync or recovery—explicitly deferred to Phase 2 “real DB” in `PLAN.md`.
- **Jest + RTL + Playwright:** Common React/Next stack; RTL tests behavior not implementation; Playwright covers the full generate/refine/save journey. Tradeoff: dual harness maintenance vs. speed—acceptable for MVP quality bar in `PLAN.md`.
- **Vercel:** First-class Next.js hosting, PR previews, simple env/secret handling. Tradeoff: platform coupling—acceptable for Phase 1 velocity.

## “Why not X” (major alternatives)

| Instead of… | Why not for Phase 1 |
|-------------|---------------------|
| **Remix / Astro / plain React SPA** | Remix/Astro are strong but **Next.js** matches the planned “app + route handlers + Vercel” path with less architectural debate; a pure SPA adds separate API hosting decisions. |
| **CSS Modules / CSS-in-JS / component library-only** | **Tailwind** matches speed and token-style consistency for a small team; can still wrap primitives in components. |
| **Anthropic / Google / multi-provider from day one** | **One provider** keeps prompts, evals, and error handling simple; multi-provider belongs after TC-040 patterns exist. |
| **Postgres / Supabase / Firebase in MVP** | **Overkill** for “save trip” before auth and product validation; adds schema, backups, and compliance surface early. |
| **Cypress instead of Playwright** | **Playwright** is a reasonable default for cross-browser E2E in 2026-era stacks; either works—team standardized on Playwright. |
| **Netlify / Fly / raw Node on a VM** | **Vercel** is the chosen host; others are valid but add friction vs. Next+Vercel defaults. |
